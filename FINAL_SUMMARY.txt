      http://localhost:8000/api/process

API - Generate draft:
    curl -X POST http://localhost:8000/api/draft \
      -H "Content-Type: application/json" \
      -d '{"doc_ids":["uuid"],"draft_type":"notice_summary"}'

API - Submit edits:
    curl -X POST http://localhost:8000/api/edit \
      -H "Content-Type: application/json" \
      -d '{"draft_id":"uuid","edited_content":"..."}'

API - List resources:
    curl http://localhost:8000/api/documents
    curl http://localhost:8000/api/drafts
    curl http://localhost:8000/api/rules

================================================================================
🧠 HOW THE LEARNING WORKS
================================================================================

When you edit a draft, the system:

1. Compares original vs edited version
2. Identifies what changed (line-by-line)
3. Sends diff to Claude for pattern extraction
4. Stores learned rules by draft type
5. Applies rules to future drafts of same type

Example rules learned:
- "Always include the contract date in the first paragraph"
- "Use 'the Company' instead of full name after first mention"
- "Add 'Next Steps' section at end of memos"
- "Prefer bullet points for obligations"

These rules improve your system over time!
